Behavior Tree - 概述

概念​

行为树（behavior tree）是用来实现角色复杂行为的工具，它具有以下特征：

  1. 行为树是树：执行时从根节点开始按照指定的顺序遍历，直到到达终结状态。
  2. 叶子结点都是可执行的节点：叶子结点会进行具体的操作，可以是一个简单的检测操作，也可以是一个更复杂的行为操作，节点会返回状态信息（成功/失败/运行中）。
  3. 内部节点控制树的遍历：内部节点会根据子节点返回的状态信息，按照特定的规则确定下一个执行的节点。

行为树常用来做任务或状态管理，最先也最多应用于游戏领域，后来引入到机器人等需要规划决策的领域。

基本原理​

通过一个例子来介绍：

  这是我们为一个士兵定义的一棵行为树（先不必管这些节点是干嘛的）。可以看到这是一个树形结构，有根节点，有分支，而且子节点个数可以任意。根节点有三个分支，分别是巡逻(Patrol)、攻击(Attack)、逃跑(Retreat)，这三个分支可以看成是士兵的三个行为方向。当然，如果有更多的行为，我们可以继续为根节点添加新的分支。当我们要决策这个士兵要做什么样的行为时，我们就会自顶向下地，通过一些条件来搜索这棵树，最终确定需要做的行为（叶子结点），并且执行它。这就是行为树的基本原理。
值得注意的是，我们标识的三大行为其实并不是真正的决策结果，它只是一个类型，来帮助我们了解这个分支是属于这类的。而真正的行为都在叶子结点上，一般是行为节点。

行为树的组成​

0.触发（Tick）​

行为树节点的一次触发称为一次tick，会返回成功(success)、失败(failure)、运行中(running)的状态信息给它的父节点。

Tips：一个节点，与它连接着的上一级节点称为它的父节点，与它连接着的下一级节点称为它的子节点。

1.节点运行状态​

每个节点一般有两种运行状态，并会向它的父节点返回自己的运行状态信息

  1. 运行中：该行为还在处理中，返回运行中（Running）
  2. 完成：该行为处理完成，返回成功（Success）或失败（Failure）

2.根节点（root）​

  即最顶端的节点，是行为树的起点，只有一个。所有行为树都有root节点。当root节点执行完成，即行为树执行完成后，会重新开始下一次执行，类似于循环。

3.执行节点（Execution Node）​

行为树的叶子结点（即行为树的枝节末端），可以是条件节点或行为节点，这些节点才是我们真正通过行为树决策出来的结果。
3-1.条件节点（Condition Node）
条件节点会在一次tick后立即返回成功或失败的状态信息，用于简单的判断，如判断某目标是否在视野内。
3-2.行为节点（Action Node）
相比于条件节点，执行行为节点可以是经过多级tick，直到到达终结状态的过程。

  通常我们需要定义不同的行为节点，但在行为树上行为节点是可以复用的。比如“移动”，在“巡逻”分支上需要用到，在“逃跑”分支上也会用到。这种情况下，我们可以复用这个节点。

4.控制节点（Control Node）​

  控制节点是行为树的内部节点，它们定义了遍历其子节点的方式。控制节点的子节点可以是执行节点，也可以是控制节点。

  控制节点分为复合节点和装饰器节点两类。复合节点可以有任意数量的子节点，而装饰器节点（Decorator）只能有一个子节点，用来对子节点的行为进行自定义修改。一般来说，常用的是三种复合节点：序列（Sequence）、选择（Selector）、并行（Parallel）。本节只介绍这三种节点。

  1. 序列节点：按顺序执行子节点，直到其中一个子节点返回失败状态或全部子节点返回成功状态
  2. 选择节点：按顺序执行子节点，直到其中一个子节点返回成功状态或所有子节点返回失败状态
  3. 并行节点：并行执行全部子节点

Tips：“按顺序”指当前一个节点达到完成状态，即返回成功或失败后，再运行下一个节点。不同的选择节点有不同的顺序，我们在下一节讨论。

控制节点图示，依次为序列、选择、并行：

控制节点是行为树的精髓所在。从控制节点上，我们可以看出整个行为树的逻辑走向。所以，行为树的特点之一就是逻辑的可见性。
除了上述几种控制节点外，我们还可以为行为树定义各种各样的控制节点，下一节将作扩展。

5.前提（Precondition）​

  看到这里想必你会产生疑问，如何从子节点中做选择呢？选择的依据是什么？这里引入另一个概念，一般称之为前提。每一个节点，不论是行为节点还是控制节点，都包含一个前提部分。
  前提提供了选择的依据，它包含了进入这个节点或者说选择这个节点的条件。当我们使用选择节点时，它依次测试每一个子节点的前提，如果满足，则选择此节点。由于我们最终返回的是某个叶子结点，所以，当前行为的“总前提”就可以看成是
当前行为的前提 && 父节点的前提 && 父节点的父节点的前提 && ······ && 根节点的前提。

Tips：根节点一般不设前提，直接返回true。

行为树就是通过执行节点、控制节点以及各个节点的前提，把整个AI的决策逻辑描述了出来。
